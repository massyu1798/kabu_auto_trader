クイックリファレンス

cd C:\kabu_auto_trader

最新をpull	
git pull origin main

タグ一覧を確認	
git fetch --all --tags && git tag -l

タグ指定で切替	
git checkout tags/v1.0.0

元のブランチに戻る	
git checkout main

変更を push	
git add .
git commit -m "メッセージ"
git push origin main

タグを作成＆push	
git tag -a v1.0.0 -m "メッセージ"
git push origin v1.0.0

Latest Release に設定	
gh release create v1.0.0 --title "v1.0.0" --notes "説明" --latest

mainを特定のブランチに戻す形で変更
git checkout main
git pull origin main
git reset --hard e1a0108d7dbbcaa4143b1f3a69ba39a92425ff07
git push --force origin main
　※git reset --hard の引数は ブランチ名 または コミットSHA を指定します
　　ブランチ名: origin/copilot/integrate-overnight-gap-strategy
　　または SHA: e1a0108d7dbbcaa4143b1f3a69ba39a92425ff07

特定のファイルだけ取得する方法
# まずリモートの最新情報を取得
git fetch origin
# 特定のファイルだけリモートの内容で上書き
git checkout origin/main -- path/to/file.txt
# 複数ファイルも指定可能
git checkout origin/main -- path/to/file1.txt path/to/file2.txt


更新の有無にかかわらず、すべてのファイルを強制的に pull する方法
# リモートの最新情報を取得
git fetch origin
# ローカルのブランチをリモートの状態に強制的にリセット
git reset --hard origin/main
⚠️ 注意: git reset --hard はローカルの未コミットの変更をすべて失います。


特定のファイルだけ add → commit → push
# 特定のファイルだけステージング（add）する
git add path/to/file1.txt path/to/file2.txt
# そのファイルだけをコミット
git commit -m "特定のファイルだけ更新"
# push する
git push origin main
