月曜日からのプログラム実行手順
1. 事前確認（日曜日のうちに）
プログラムが正しく設定されているか最終チェックします。

APIパスワードの確認: config/live_config.yaml の password 欄に、kabuステーションで設定した「APIパスワード」が正しく入力されているか。
動作モードの確認: config/live_config.yaml の paper_trade が true になっているか。（最初の数日間は true（発注しないモード） で動作を確認することを強くお勧めします）
資金設定の確認: initial_capital が 7000000 になっているか。

2. 当日朝の準備（08:30 ～ 08:55）
市場が始まる前に環境を整えます。

kabuステーションを起動・ログイン:
通常通りログインします。
重要: 設定で「APIを有効にする」にチェックが入っていることを確認してください。
コマンドプロンプトを起動:

cd C:\kabu_auto_trader

 を実行してフォルダへ移動。

venv\Scripts\activate

 を実行して仮想環境を有効化。（左端に (venv) と出ればOK）

3. プログラムの起動（08:55 ～ 09:00）
いよいよ実行します。

起動コマンドを入力:

python main_live.py

起動メッセージを確認:
✅ トークン取得成功 と表示されるか。
市場が開くまで待機します... と表示されれば準備完了です。
9:00になると自動的に株価の取得と監視が始まります。

4. 市場時間中の監視（09:00 ～ 15:30）
プログラムが動いている間、以下の点に注意してください。

PCのスリープ禁止: PCがスリープすると通信が切れて止まってしまいます。電源設定で「スリープしない」にしておいてください。
kabuステーションを閉じない: APIの親玉であるkabuステーションを閉じると、プログラムもエラーで止まります。
ログの確認:
🔍 [10:00:00] シグナル判定中... と定期的に出ているか。
シグナルが出た際、📝 [PAPER] BUY ... と表示されるかを確認。

5. 市場終了後（15:30以降）
プログラムが1日の仕事を終えます。

日次サマリーの確認:
15:30を過ぎると、その日のトレード数や仮想損益の合計が表示されます。
プログラムの停止:
コマンドプロンプトで Ctrl + C を押して終了させます。
kabuステーションの終了:
プログラムを止めた後であれば、kabuステーションを閉じても大丈夫です。

🚨 緊急時の停止方法
もしプログラムの動きがおかしい、あるいは意図しない発注（本番モード時）が怖いと感じた場合は、迷わず以下の操作をしてください。

プログラム（コマンドプロンプト）を閉じる（右上の×ボタンでOK）。
kabuステーションを終了させる。
これでAPI経由の発注は物理的に遮断されます。
kabuステーションまたはスマホアプリから手動で決済する。
万が一ポジションが残ってしまった場合は、証券会社のツールから手動で返済注文を出してください。
